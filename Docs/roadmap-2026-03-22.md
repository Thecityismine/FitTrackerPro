## P0 - Ship-Critical Polish

### 1. Language Consistency

Goal: make labels short, human, and consistent across the app.

Standards:

- `LBS LIFTED` -> `Total Volume`
- `lbs vol.` -> `Volume`
- `Analyze My Progress` -> `View Insights`
- `Analyze Your Week` -> `Weekly Insights`

Rules:

- prefer short labels
- avoid duplicate meanings with different wording
- use the same term for the same concept everywhere

Likely files:

- `src/pages/CalendarLog.jsx`
- `src/pages/Dashboard.jsx`
- `src/pages/WorkoutPage.jsx`
- `src/pages/Profile.jsx`
- `src/pages/BodyMetrics.jsx`
- `src/pages/Routines.jsx`

Definition of done:

- no mixed variants of `Volume`, `Total Volume`, `lbs lifted`, `time logged`, or `Analyze` CTAs unless intentionally different

### 2. CTA System Unification

Goal: define only two CTA classes across the app.

Primary CTA examples:

- `Start Workout`
- `Continue Workout`
- `View Insights`
- `Weekly Insights`

Secondary CTA examples:

- `Log Manually`
- `Repeat Workout`

Rules:

- primary actions use the blue gradient treatment
- secondary actions use outlined or subdued treatment
- remove one-off CTA styles where possible

Definition of done:

- no screen mixes large premium CTAs with unrelated text-link actions for equivalent actions

### 3. Empty State Upgrade

Goal: inactive states should still feel intentional and motivating.

Direction:

- replace flat absence text with helpful prompts
- make each empty state tell the user what to do next
- keep tone encouraging, not generic

Example:

- `No photo last month` ->
  `Add your first check-in`
  `Track real progress month to month`

Definition of done:

- no empty panel feels blank, unfinished, or abandoned

### 4. App Store Readiness

Must-have before screenshots or submission:

- no placeholder text
- no dead buttons
- no incomplete flows
- no obvious loading stalls
- basic error handling on critical actions
- full start-to-finish workout flow works cleanly

Risk:

- Apple rejection if the app feels incomplete or broken even once during review

## P1 - High-Impact Premium Polish

### 5. Micro Animations

Goal: subtle motion that makes the app feel premium without becoming flashy.

Add:

- calendar day tap: soft scale and glow
- CTA press: slight shrink and bounce
- cards: fade or slide in
- progress bars: animate on load

Definition of done:

- motion feels responsive and deliberate, not noisy

### 6. Data Hierarchy Consistency

Goal: make the main number visually dominant and the context secondary.

Rule:

- big number = main metric
- small label = context

Preferred pattern:

- `32.8k`
- `Total Volume`

Instead of:

- `32.8k lbs lifted`

Definition of done:

- stats across dashboard, progress, and workout summary follow the same hierarchy

### 7. Icon Consistency

Goal: icons should feel like one system.

Checklist:

- same stroke width
- same card padding
- same vertical alignment
- same visual weight

Definition of done:

- no icon feels heavier, denser, or misaligned compared to the rest

### 8. Emotional Hook Per Screen

Goal: pair data with motivation so users keep coming back.

Examples:

- `Strong week`
- `You're ahead of pace`
- `Keep the streak alive`

Definition of done:

- each main screen has at least one short motivational line that supports the data

## P2 - Targeted UX Improvements

### 9. Progress Page Insight Order

Goal: insight first, then supporting data.

Change:

- move comparison text like `+31% vs last session` above the workout card

Reason:

- users process the takeaway faster when the insight appears before the detail

### 10. Recovery Page Interaction

Goal: make the recovery heatmap functional, not just visually strong.

Ideas:

- tap chest -> show exercises to train
- tap legs -> suggest workout

Definition of done:

- at least one meaningful tap interaction exists on the body map
